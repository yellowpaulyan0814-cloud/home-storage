## 一、项目概述

### 核心目标

一个纯前端网页应用，帮助家庭记录物品存放在哪个柜子里，支持快速搜索、可视化地图定位。

### 关键约束

- 完全离线可用（双击 index.html 运行）
- 不依赖任何后端服务器、Node.js、Python、NPM
- 纯 HTML + CSS + JavaScript（无 React/Vue/TypeScript）
- 模块化设计，便于长期维护扩展
- 支持 PC / 平板 / 手机
- 数据存储优先使用 IndexedDB，云端同步为可选增强

### 访问方式

- **本地**：双击 index.html
- **线上**：GitHub Pages（`https://{用户名}.github.io/{仓库名}/`）
- **密码保护**：首次访问需输入家庭密码（sessionStorage 存储，关闭标签页后需重新输入）

## 四、功能模块详解

### 4.1 密码门（app.js）

- 应用启动先检查 `sessionStorage['hs_auth_passed']`
- 未通过：渲染密码输入界面（巨大密码框 + 回车验证）
- 通过：正常启动路由
- 密码硬编码为 `const FAMILY_PASSWORD = 'yzyz2024'`，可修改

### 4.2 路由系统（router.js）

基于 URL Hash 的 SPA 路由：

| 路由模式 | 对应页面 | 参数说明 |
|----------|----------|----------|
| `/` 或 `/search` | 搜索首页 | `?q=关键词` 可预设搜索词 |
| `/add` | 新增物品 | `/add/:id` 为编辑模式 |
| `/map` | 柜子地图（房间选择） | — |
| `/map/:room` | 房间柜子布局 | `?cabinet=X&level=Y&highlight=ID` 搜索跳转 |
| `/map/:room/:cabinet` | 柜子详情（各层物品） | — |
| `/map/:room/:cabinet/:level` | 柜子详情+指定层高亮 | `?highlight=ID` |
| `/settings` | 设置页面 | — |

路由解析逻辑：`parseHash()` 将 `#/map/K?cabinet=K-KD03&highlight=xxx` 解析为 `{ path, parts: ['map','K'], query: { cabinet: 'K-KD03', highlight: 'xxx' } }`。

### 4.3 IndexedDB 封装（db.js）

数据库名称：`HomeStorageDB`，版本：1，Object Store：`items`（keyPath: 'id'）。

索引：`name`、`room`、`cabinet`、`cabinetLevel`（复合索引 `[cabinet, level]`）、`createTime`、`updateTime`。

核心函数：

| 函数 | 说明 |
|------|------|
| `openDB()` | 打开/初始化数据库（单例模式） |
| `addItem(data)` | 新增物品，自动生成 id + 时间戳 |
| `getItem(id)` | 按ID查单条 |
| `updateItem(id, data)` | 更新物品，自动刷新 updateTime |
| `deleteItem(id)` | 删除物品 |
| `getAllItems()` | 获取全部物品（按 updateTime 倒序） |
| `searchItemsFromDB(query)` | 模糊搜索（先全量读入内存再匹配） |
| `getItemsByCabinet(cabinet)` | 按柜子ID查所有物品 |
| `getItemsByCabinetLevel(cabinet, level)` | 按柜子+层查 |
| `getItemsByRoom(room)` | 按房间查 |
| `exportData()` | 导出全部数据为 JSON 字符串 |
| `importData(jsonStr)` | 导入 JSON（用 ID 去重，不覆盖已有数据） |
| `clearAllData()` | 清空全部数据 |
| `getStats()` | 统计信息（总量、各房间数量、最近更新5条） |

每次写操作（add/update/delete/clear/import）成功后会调用 `cloudPushAfterChange()` 自动推送到云端。

### 4.4 模糊搜索算法（utils.js）

`fuzzyMatch(query, target)` 实现类 Sublime/VSCode 的模糊匹配：
1. 完全匹配 → 100分
2. 子串包含匹配 → 80分 + 长度比例加分
3. 逐字符按序匹配 → 50分 + 连续匹配加分 + 位置靠前加分
4. 所有字符都匹配才算命中，否则0分

`searchItems(query, items)` 对物品的 name、box、remark 三个字段分别模糊匹配，取最高分，按得分降序排列。

### 4.5 柜子 SVG 布局系统（config.js + map.js）

#### 配置结构

每个房间的柜子布局定义在 `CABINET_LAYOUTS` 对象中：

```javascript
K: {
    type: 'svg',           // 'svg' 或 'photo'
    viewBox: '0 0 1000 700',
    cabinets: [            // 柜子位置数组
        { id: 'K-KL01', x: 15, y: 20, w: 65, h: 132, rx: 6, groupLabel: '左侧' },
        // ...
    ],
    decorations: [         // 装饰元素（文字、虚线框等）
        { type: 'rect', x: 170, y: 145, w: 390, h: 210, rx: 10, label: '📺 电视区域' },
        { type: 'text', x: 35, y: 200, label: '空', fontSize: 12, color: '#ddd' }
    ]
}
```

#### 渲染流程

`renderRoomMap()` → 获取每个柜子的物品计数 → 根据 `layout.type` 选择渲染方式：
- `'svg'`：调用 `renderSVGMap()` → 生成 `<svg>` 元素，每个柜子为一个 `<g>` 组，包含 `<rect>`（柜门）+ `<text>`（编号+名称）+ 物品数量角标。hover 显示悬浮提示，click 进入柜子详情。
- `'photo'`：调用 `renderPhotoMap()` → 图片背景 + 百分比定位的 `<div>` 热区覆盖层。

#### 搜索联动地图

从搜索页面点击"在地图中查看" → 导航到 `/map/:room?cabinet=X&level=Y&highlight=ID` → `renderRoomMap()` 检测到 query 参数 → 展示完整房间地图 + 目标柜子高亮（黄色闪烁边框）+ 顶部横幅提示"物品'XXX'在 柜子名称 · 第N层"。

### 4.6 新增物品：双模式柜子选择（add.js）

**列表模式**：房间下拉 → 柜子下拉（级联过滤，按分组显示）→ 层数下拉（级联过滤）。

**地图模式**：选择房间后 → 渲染该房间的 SVG 迷你地图（`refreshMiniMap()`）→ 点击柜门直接选中 → 自动更新层数下拉。迷你地图使用自适应 viewBox（自动裁剪柜子区域的紧凑包围盒）。

两种模式可通过按钮切换，值双向同步。

### 4.7 物品详情弹窗（detail.js）

`showItemDetail(id)` 从底部滑入面板：
- 显示物品名称、位置链（房间 → 柜子 → 层）、收纳盒、备注、创建/修改时间
- 操作按钮：在地图中查看（搜索联动）、编辑（跳转 `/add/:id`）、删除（确认对话框）

### 4.8 云同步（cloud.js）

#### GitHub Gist 方案

- Gist ID：`51e0a995c957ca06fe2daf2f2c526fe8`
- Token：XOR 加密硬编码在 `cloud.js` 中
- 写入：`PATCH /gists/:id` → 整个 items 数组推送到 Gist
- 读取：`GET /gists/:id` → 读取 Gist 中的 items 数组

#### 同步策略

- **写入触发**：每次 IndexedDB 写操作（add/update/delete/clear/import）后自动调用 `cloudPushAfterChange()` 异步推送到 Gist
- **读取触发**：应用启动时（app.js 初始化1秒后）、每次搜索前（search.js doSearch 前）
- **双向合并**：以 `updateTime` 为准，保留每条物品的最新版本

#### 安全性

- Token 在代码中使用 XOR 加密存储，GitHub Secret Scanner 无法检测
- 网页有密码门保护，陌生人无法使用
- Gist 为私有 Gist，仅持有 Token 者可读写
- iOS Safari/Linux 环境因 atob 限制可能降级为内置明文备份 Token

### 4.9 设置页面（settings.js）

- 数据统计：总物品数 + 各房间数量（卡片展示）
- 最近更新列表（点击可查看详情）
- 导出 JSON（下载为 `家庭收纳备份-YYYYMMDD.json`）
- 导入 JSON（文件选择器，追加模式，通过 ID 去重）
- 清空全部数据（两次确认对话框）
- 云同步状态（检查云端 + 立即同步按钮）
- 关于信息（版本号、数据存储方式）

## 七、部署与运维

### 本地开发

双击 `index.html` 即可运行。使用浏览器开发者工具查看 IndexedDB。

### 部署到 GitHub Pages

1. 创建 GitHub 仓库（Public）
2. 推送代码到 master 分支
3. Settings → Pages → Source: Deploy from a branch → Branch: master → Save
4. 等1-2分钟，访问 `https://{用户名}.github.io/{仓库名}/`

### 数据备份

- **自动备份**：云同步开启后自动备份到 Gist
- **手动备份**：设置页 → 导出 JSON → 下载到本地
- **版本备份**：`cp -r 项目目录 E:/备份-日期/`

### Token 更换

如果 GitHub Token 过期需要更换：
1. [github.com/settings/tokens](https://github.com/settings/tokens) 生成新 Token（勾选 gist 权限）
2. 用 Python 生成新的 XOR 加密数组：
```python
token = 'ghp_新Token'
key = 42
encrypted = ','.join(str(ord(c) ^ key) for c in token)
```
3. 替换 `cloud.js` 中 `_tk` 数组的值

### Git 远程 URL 格式

```
https://{用户名}:{Token}@github.com/{用户名}/{仓库名}.git
```

## 八、已知限制与后续扩展

### 已知限制

- 数据量大（>5000条）时模糊搜索可能变慢（当前全量读入内存再匹配）
- 云同步为"最后写入胜出"策略，不支持冲突解决
- iOS Safari 环境 atob 可能降级为明文 Token

### 预留扩展点

- `cloud.js` 可替换为其他后端（API 接口统一）
- 拼音搜索、别名搜索接口（在 utils.js 预留）
- PWA manifest（已预留 meta 标签）
- 照片热区模式已在 map.js 实现，随时可用
- `config.js` 中修改房间/柜子定义即可扩展新房间
